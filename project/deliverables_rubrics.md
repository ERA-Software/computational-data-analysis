# 📊 CDAP's Project: Deliverables and Assessment

CDAP's project follows a structured, **end-to-end data analysis workflow**, developed progressively across three notebook submissions.

All deliverables must be submitted as **well-documented Jupyter notebooks**, including:
- explanatory markdown cells,
- clear and reproducible code,
- interpretation of results.

---

# 📏 Rubrics by deliverable

## 🔹 Deliverable 1 — Problem framing & EDA

### 1. Problem formulation (30%)
- Clear, specific, and well-motivated question  
- Defined target and intended use  

**Levels**
- Excellent: precise, relevant, decision-oriented  
- Satisfactory: clear but generic  
- Poor: vague or unclear  

---

### 2. Data understanding & quality checks (30%)
- Dataset description (variables, types, units if relevant)  
- Identification of missing values, anomalies, inconsistencies  

**Levels**
- Excellent: systematic and insightful checks  
- Satisfactory: basic checks  
- Poor: superficial or missing  

---

### 3. Exploratory data analysis (EDA) (30%)
- Meaningful visualizations and summaries  
- Clear connection between plots and the question  

**Levels**
- Excellent: insightful, well-justified EDA  
- Satisfactory: correct but limited insight  
- Poor: descriptive or unfocused  

---

### 4. Communication & structure (10%)
- Clear notebook organization  
- Readable code and explanations  

---

## 🔹 Deliverable 2 — Data analysis & prediction

### 1. Data preprocessing (25%)
- Clear, reproducible pipeline  
- Justification of handling missing data, features, etc.  

**Levels**
- Excellent: well-justified and reproducible  
- Satisfactory: partially justified  
- Poor: unclear or ad hoc  

---

### 2. Method selection & implementation (25%)
- Appropriate baseline method  
- Correct and clear implementation  

**Levels**
- Excellent: well-justified and correctly implemented  
- Satisfactory: reasonable but limited  
- Poor: inappropriate or incorrect  

---

### 3. Validation & evaluation (30%)
- Appropriate validation strategy  
- Use of relevant metrics  
- Basic error analysis  

**Levels**
- Excellent: rigorous and clearly explained  
- Satisfactory: basic evaluation  
- Poor: weak or missing  

---

### 4. Interpretation of results (10%)
- Initial insights from model outputs  
- Connection to the original question  

---

### 5. Communication & reproducibility (10%)
- Clean structure and readable notebook  

---

## ✅ Key Principle

Each stage builds on the previous one.  
Early submissions focus on **understanding the problem and the data**,  
while later submissions emphasize **modeling, evaluation, and decision-oriented insight**.
